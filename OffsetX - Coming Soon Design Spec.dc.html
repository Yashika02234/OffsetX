§ 02 — GLOBAL DESIGN SYSTEM

Design language

The visual grammar is built on restraint. Every element earns its place. Color is functional — teal signals action or emphasis, never decoration. The grid is the only ornament. Whitespace is a design tool, not wasted space.

2.1 — COLOR SYSTEM
TOKEN SWATCH HEX USAGE
--color-bg
#080E0B Global page background. The foundation of everything.
--color-surface
#0D1A14 Cards, counter strip, raised surfaces. 0.5px teal-dim border.
--color-surface-2
#0A1410 Nav bar behind blur. Slightly darker than surface.
--color-accent
#1AFFE0 Primary accent. Used for: wordmark, "trust layer" in H1, primary button fill, counter numbers, eyebrow labels. Never as background texture.
--color-border
rgba(26,255,224,0.1) All borders — cards, nav divider, table rules. Never full-opacity teal on borders.
--color-text-primary
#DFF2EA H1, H2, H3, card titles. Off-white with a green cast — not pure white, which feels cold.
--color-text-body
#C8E6DA All body copy. Slightly dimmer than primary text — creates effortless hierarchy.
--color-text-muted
#7AB8A0 Captions, secondary labels, card descriptions. One step further from primary.

Border rule: The page brief explicitly states every border must use rgba(26,255,224,0.1) — the ghost of the teal accent. This creates the impression of a grid made of carbon measurement lines without any visual noise at full opacity.

§ 06 — PREMIUM DETAILS

The details that separate good from excellent

These are the micro-decisions that the user never consciously notices — but that make the page feel premium, considered, and trustworthy. Each one is deliberately small.

01
Scroll-linked nav border
Nav bottom-border fades from transparent to rgba(26,255,224,0.1) after 10px scroll. 300ms ease. Signals context without visual shock.
02
H1 word-by-word reveal
On page load, each word of the H1 fades up individually with 40ms stagger. Reads as intentional precision, not a template transition.
03
Eyebrow label blinking cursor
On first load only, the eyebrow label "types in" with a blinking cursor. After 1 pass, cursor disappears. Signals a live terminal without being gimmicky.
04
Button arrow nudge on hover
The "→" in "JOIN EARLY ACCESS →" translates +3px on button hover, 150ms ease. Implies momentum.
05
Counter suffix animation
The "+" suffix on stats (20+, 12+, 30+) fades in 200ms after the number finishes counting. Not simultaneous — a beat of pause, then the plus sign.
06
Focus ring system
All interactive elements: 2px outline #1AFFE0, 2px offset on focus. Never the browser default blue. Keyboard users get the same premium experience.
07
Email input focus glow
On focus, email input border transitions from rgba(26,255,224,0.15) to rgba(26,255,224,0.4). Very subtle box-shadow: 0 0 0 3px rgba(26,255,224,0.06). 200ms ease.
08
Research card link sweep
On card hover, a 1px bottom border sweeps left-to-right with scaleX 0→1, transform-origin: left, 250ms ease. Reads as an underline being drawn in real time.
09
Reduced-motion safety
@media (prefers-reduced-motion: reduce) disables all transitions, animations, counter animations. Content renders instantly in place.
10
Page-load progress bar
A 1px teal progress bar sweeps across the very top of the viewport on page load (above the nav). Opacity fades to 0 on completion. 800ms total.
11
Smooth anchor scrolling
scroll-behavior: smooth + scroll-margin-top: 72px on all section anchors (clears the fixed nav height + 16px breathing room).
12
Wordmark hover state
On nav wordmark hover, a very subtle white flash (brightness 1.05) fades in then out over 200ms. Not a color change — a shimmer. Signals it's a link without breaking the palette.
13
Section label fade-in timing
When a section scrolls into view, the eyebrow label fades in at t=0ms, then the H2 fades at t=80ms. The sequence mirrors how a document renders — metadata before content.
14
No animation on second visit
sessionStorage flag prevents hero entrance animation from replaying on refresh within the same session. Only triggers once per visit.
15
Selection color
::selection { background: rgba(26,255,224,0.2); color: #DFF2EA; }. Highlighted text matches the brand. A detail only obsessives notice — and obsessives are the audience.
16
Input placeholder color
Email input placeholder: rgba(200,230,218,0.3). Not the browser default grey. Matches the page's tonal range even in unfilled state.
17
Left accent bar parallax
The 4px left accent bar very subtly shifts: at scroll 0 it is at rgba(26,255,224,0.12); at full-page bottom it dims to rgba(26,255,224,0.06). Achieved with a single scroll listener. 60fps smooth.
18
Favicon and browser title
Favicon: white "OX" on #080E0B background, 32×32px SVG. Browser tab title: "OffsetX — Carbon Intelligence Infrastructure". Meta description: the H1 subheading text, verbatim.
§ 07 — DESIGN DIRECTOR CRITIQUE

Honest self-review

Pretending I'm sitting in the design review at Stripe or Linear. What would we push back on?

WEAKNESSES
No above-the-fold social proof

The hero section has no logos, quotes, or validation signals. For a B2B product targeting risk-averse corporate buyers and investors, the absence of any credibility anchor in the hero is a conversion risk. "Building the trust layer" is a claim — nothing in the hero substantiates it before the scroll.

The problem section assumes domain knowledge

Terms like "CBAM", "VCM", "MRV", "Article 6", and "UNIC" are used without any contextual tooltip or glossary. For investors outside the carbon space (common in climate tech VC), this creates friction. A single clarifying line under each card header ("CBAM = Carbon Border Adjustment Mechanism") would convert more without breaking the aesthetic.

The email capture is under-motivated

Asking for an email before articulating the specific value of early access is weak. "Be among the first organisations" is generic. The section needs a specific hook: what do early access participants get? A beta, a briefing, a data preview? The copy must earn the email address.

No founder/team signal

For a pre-product company, the founding team's credibility is a core asset. The brief mentions "TALK TO THE FOUNDER" but provides no context on who the founder is. A single line of founder bio — industry background, previous role — would dramatically increase conversion among investors.

What We're Building is undifferentiated at first glance

The five-item list reads as features, not a product. It doesn't communicate the integrated platform vision — how the five capabilities connect. A single line above the list articulating the architecture (e.g. "One connected platform. Five capabilities.") would reframe the section from a feature list to a system vision.

RECOMMENDED IMPROVEMENTS
Add a "backed by" or "supported by" signal

Even a single line — "Funded by [investor name]" or "Supported by [accelerator]" — positioned discreetly below the CTAs would shift the hero's trust signal from 0 to something. If no investors yet, a press mention or institutional partnership would serve the same function.

Add a scroll progress indicator

A 1px teal line that grows from 0 to 100% width at the top of the viewport as the user scrolls the page. Reinforces the document-like reading experience and gives spatial orientation on a longer page.

Add a quote section between counters and research

One or two anonymised expert quotes — rendered large, editorial, dark-panel — between the counter strip and research grid would add human texture to what is currently an entirely data-driven narrative. Even without attribution, the quotes signal depth of discovery.

Active nav highlighting

As the user scrolls past each section, the corresponding nav link should update to color #DFF2EA. Implemented with IntersectionObserver per section. Gives spatial awareness and makes the nav feel intelligent.

ENTERPRISE PREMIUM ADDITIONS
OpenGraph / Twitter Card metadata

When shared on LinkedIn (the primary channel for this audience), the link preview must show a branded card: dark bg, wordmark, H1 excerpt. This is the first impression most enterprise visitors have of the page.

Accessibility: ARIA landmarks

Each section should be wrapped in a semantic landmark (<main>, <section aria-label>, <nav>). Enterprise buyers often run accessibility audits before vendor approval.

Plausible / privacy-first analytics

Add Plausible.io (1KB, cookieless, GDPR-native). Captures which CTA converts, scroll depth, source of traffic. Critical for knowing which LinkedIn posts drive email captures.

Lighthouse score target: 95+

Performance, accessibility, best practices, SEO. Single HTML file with no external JS frameworks means this is achievable. Space Grotesk must use font-display: swap. All images must have alt text.
